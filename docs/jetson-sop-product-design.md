# JETSON 设备包装 SOP 行为检测系统产品设计文档

## 1. 产品概述

### 1.1 产品名称

JETSON 设备包装 SOP 行为检测系统

### 1.2 产品定位

本产品是一套基于 AI 视觉的工业 SOP 合规检测系统，用于在设备包装工位实时识别操作员动作、物料状态和包装流程进度，判断操作是否按标准顺序完成。

系统通过工位摄像头采集俯拍画面，结合目标检测、手部关键点识别和 SOP 状态机，自动判断包装流程是否存在漏放、错放、跳步、顺序错误或超时等异常，并在界面中实时展示检测结果。

### 1.3 目标用户

- 工厂产线管理人员
- 质检人员
- 包装工位操作员
- 工业视觉系统集成商
- 边缘 AI 设备方案商

### 1.4 核心价值

- 降低人工巡检成本
- 减少包装漏件、错件和跳步骤问题
- 将 SOP 执行从事后抽检转为实时校验
- 形成包装过程的可追溯记录
- 支持 Jetson 等边缘设备本地实时推理

## 2. 视频演示能力总结

视频演示的是一个单工位包装检测后台。界面左侧为实时摄像头画面，右侧为 SOP 步骤状态表和检测日志，底部为步骤状态条。系统会对操作员双手、包装盒、保护海绵、纸质资料、Jetson 设备等目标进行识别，并按照预设流程推进每一步状态。

### 2.1 实时工位视频检测

系统左侧展示工位俯拍画面，摄像头覆盖桌面包装区域。

画面中实时绘制以下视觉元素：

- 工位检测区域 ROI
- 左手 / 右手检测框
- 手部关键点骨架
- 包装盒检测框
- 海绵、纸质资料、Jetson 设备等物料检测框
- 目标类别与置信度

### 2.2 SOP 步骤状态识别

右侧表格展示 SOP 步骤状态，表头包含：

| 字段 | 说明 |
| --- | --- |
| No | 步骤编号 |
| 步骤 | 当前 SOP 步骤名称 |
| 结果 | Idle / Running / Done |
| 说明 | 等待执行 / 正在检测 / 已完成 |

视频中展示的 SOP 步骤如下：

| No | 步骤 | 说明 |
| --- | --- | --- |
| 1 | 开口空盒检测 | 检测桌面上是否存在打开状态的空包装盒 |
| 2 | 放入保护海绵 | 检测保护海绵是否放入包装盒 |
| 3 | 放入纸质资料 | 检测说明资料、三包凭证等纸质资料是否放入 |
| 4 | 放入 Jetson 设备 | 检测 Jetson 设备是否放入包装盒 |
| 5 | 合盖完成 | 检测包装盒是否完成合盖 |

### 2.3 状态流转

每个 SOP 步骤有三种核心状态：

| 状态 | 含义 | UI 表现 |
| --- | --- | --- |
| Idle | 等待执行 | 深色 / 灰蓝 |
| Running | 正在检测 | 蓝色 |
| Done | 已完成 | 绿色 |

流程按固定顺序推进：

1. 检测开口空盒
2. 放入保护海绵
3. 放入纸质资料
4. 放入 Jetson 设备
5. 合盖完成
6. 全部完成后输出 OK

### 2.4 当前步骤与检测日志

界面右下角展示实时检测日志，用于说明当前流程判断依据。

日志信息包括：

- 当前步骤
- 检测数量
- 检测耗时
- 有效目标
- 低分目标
- 识别结果
- 流程是否完成

示例：

```text
当前步骤：先放入任一纸质资料
检测数量：0
检测耗时：2
有效目标：开口盒1，保护海绵1
低分目标：0
识别结果：False
三包流程完成：False
```

完成后示例：

```text
当前步骤：全部完成
检测数量：8
有效目标：闭合盒1
识别结果：True
三包流程完成：True
```

### 2.5 顶部运行状态栏

顶部状态栏用于现场人员快速判断任务执行情况。

视频中可见字段包括：

- 项目：JETSON 设备包装 SOP
- 作业员：张三
- 模式：常规顺序
- 状态：运行 / 完成
- 耗时：00:42
- 结果：检测中 / OK

### 2.6 底部步骤状态条

底部以卡片形式展示每个步骤的当前状态。每个卡片包含：

- 步骤名称
- 当前状态：Idle / Running / Done

该区域与右侧 SOP 表格同步变化，用于增强现场可读性。

## 3. 业务流程设计

### 3.1 正常流程

```text
开始任务
  ↓
检测开口空盒
  ↓
放入保护海绵
  ↓
放入纸质资料
  ↓
放入 Jetson 设备
  ↓
合盖完成
  ↓
输出 OK
  ↓
保存检测记录
```

### 3.2 异常流程

系统需要识别并处理以下异常：

- 前一步未完成时执行后续步骤
- 目标物缺失
- 放入错误物料
- 物料未进入指定 ROI 区域
- 当前步骤超时
- 检测置信度低于阈值
- 手部遮挡导致连续多帧无法确认

异常处理建议：

1. 当前步骤保持 Running 或标记 NG。
2. 界面展示异常原因。
3. 可触发声光报警或弹窗提示。
4. 保存异常关键帧和日志。
5. 支持人工复核或重新检测。

## 4. 功能需求

### 4.1 摄像头接入

#### 功能说明

接入工位摄像头，实时采集包装操作画面。

#### 需求

- 支持 USB 摄像头
- 支持 RTSP / ONVIF 网络摄像头
- 支持 720p / 1080p 分辨率
- 支持帧率配置
- 支持实时预览
- 支持摄像头断线重连
- 支持画面镜像、旋转和裁剪

### 4.2 ROI 检测区域配置

#### 功能说明

设置工位有效检测区域，减少背景干扰。

#### 需求

- 支持在视频画面上绘制矩形或多边形 ROI
- 支持保存 ROI 配置
- 支持每个工位独立配置 ROI
- 仅当目标进入 ROI 时参与 SOP 判定

### 4.3 AI 目标检测

#### 识别类别

MVP 阶段建议识别以下类别：

| 类别 | 说明 |
| --- | --- |
| left_hand | 左手 |
| right_hand | 右手 |
| open_box | 开口包装盒 |
| foam | 保护海绵 |
| paper_doc | 纸质资料 |
| warranty_card | 三包凭证 |
| jetson_device | Jetson 设备 |
| closed_box | 合盖包装盒 |

#### 检测输出

```json
{
  "class_name": "jetson_device",
  "confidence": 0.93,
  "bbox": [120, 80, 360, 280],
  "timestamp": 1778344800000
}
```

### 4.4 手部关键点识别

#### 功能说明

识别操作员左右手和手部关键点，用于判断动作交互，而不是仅凭物体静态出现判定步骤完成。

#### 需求

- 识别左右手
- 输出手部检测框
- 输出关键点骨架
- 支持手部与物料的空间关系判断
- 支持手部进入 / 离开目标区域判断

### 4.5 SOP 状态机

#### 功能说明

状态机负责把 AI 识别结果转化为业务步骤状态。

#### 状态定义

```text
Idle    等待执行
Running 正在检测
Done    已完成
NG      异常
```

#### 推进规则

- 任务开始后，步骤 1 进入 Running。
- 当前步骤目标被连续稳定检测到 N 帧后，判定 Done。
- 当前步骤 Done 后，下一步骤进入 Running。
- 非当前步骤目标可记录日志，但不直接推进流程。
- 所有步骤 Done 后，任务结果为 OK。
- 任一步骤超时或出现强异常时，任务结果为 NG。

#### 判定示例

```json
{
  "step_id": 4,
  "name": "放入 Jetson 设备",
  "required_objects": ["open_box", "jetson_device"],
  "required_hand_interaction": true,
  "stable_frame_count": 5,
  "min_confidence": 0.8,
  "timeout_seconds": 30
}
```

### 4.6 实时 UI 展示

#### 功能说明

前端实时展示摄像头画面、检测框、SOP 状态、当前日志和最终结果。

#### 需求

- Canvas 绘制检测框和关键点
- 右侧展示 SOP 表格
- 底部展示步骤状态条
- 顶部展示项目、作业员、耗时、结果
- 支持 OK / NG 高亮显示
- 支持低置信度目标提示
- 支持检测中、完成、异常等状态切换

### 4.7 检测记录保存

#### 功能说明

每次包装流程完成后生成检测记录。

#### 数据字段

| 字段 | 说明 |
| --- | --- |
| task_id | 检测任务 ID |
| sop_id | SOP 模板 ID |
| workstation_id | 工位 ID |
| operator_id | 作业员 ID |
| start_time | 开始时间 |
| end_time | 结束时间 |
| duration | 总耗时 |
| result | OK / NG |
| failed_step | 异常步骤 |
| error_reason | 异常原因 |
| snapshots | 关键帧截图 |
| logs | 检测日志 |

## 8. 技术方案

### 8.1 系统架构

```text
摄像头
  ↓
视频采集服务
  ↓
AI 推理服务
  ↓
目标检测 / 手部关键点识别
  ↓
SOP 状态机
  ↓
实时消息推送
  ↓
Web 监控界面
  ↓
日志与结果存储
```

### 8.2 推荐技术栈

#### 前端

- React / Vue
- Canvas / WebGL 绘制检测框
- WebSocket 接收实时检测结果
- 表格组件展示 SOP 状态
- ECharts 展示统计报表

#### 后端

- Python FastAPI
- WebSocket
- REST API
- PostgreSQL / MySQL
- Redis 缓存实时状态

#### AI 推理

- YOLOv8 / YOLOv10 / RT-DETR：目标检测
- MediaPipe Hands / 自训练手部关键点模型：手部识别
- ONNX Runtime / TensorRT：推理加速
- OpenCV：视频采集与图像处理

#### 边缘部署

- NVIDIA Jetson Nano / Xavier / Orin
- 本地摄像头接入
- 本地模型推理
- 局域网 Web 管理界面
- 离线缓存检测记录

## 10. 验收标准

### 10.1 基础功能验收

- 系统可稳定接入摄像头并显示实时画面。
- 系统可识别开口盒、保护海绵、纸质资料、Jetson 设备和合盖包装盒。
- 系统可展示左手、右手检测框和手部关键点。
- SOP 表格可正确显示 Idle、Running、Done 状态。
- 底部步骤状态条与右侧 SOP 表格保持一致。
- 完成全部步骤后，顶部结果显示 OK。
- 跳过步骤时，系统不能误判为完成。

### 10.2 性能验收

- 实时画面端到端延迟小于 500ms。
- 单帧推理耗时小于 100ms。
- 正常光照下目标检测准确率大于 95%。
- 连续运行 8 小时无崩溃。
- 摄像头断线后可自动重连。

### 10.3 业务验收

- 能发现漏放保护海绵、漏放资料、漏放设备等异常。
- 能记录每个包装任务的开始、结束和步骤耗时。
- 能查看每个异常任务的原因和关键帧。
- 能支持至少一个包装工位稳定运行。

## 11. 复刻开发建议

建议按以下顺序实现：

1. 先完成单工位实时视频预览和检测框绘制。
2. 接入目标检测模型，识别包装盒、海绵、资料和设备。
3. 接入手部检测，增加操作交互判断。
4. 实现固定五步 SOP 状态机。
5. 完成主监控页面，还原视频中的左画面、右表格、底部状态条布局。
6. 增加检测记录保存和异常截图。
7. 再扩展 SOP 可配置、多工位和后台管理能力。

## 12. 总结

该产品的核心不是单纯做物体识别，而是把目标检测、手部行为识别、ROI 约束和 SOP 状态机组合起来，实时判断包装动作是否按标准流程完成。

复刻开发时应优先保证以下三点：

1. 目标识别稳定，减少误检和漏检。
2. SOP 状态机严格按顺序推进，避免跳步误判。
3. UI 实时、直观，让现场人员能快速看到当前步骤、异常原因和最终结果。
